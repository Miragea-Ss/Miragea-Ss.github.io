# Atelier Agent Sources

This site is a **serious business entry point**, not a demo or play homepage.

## Campaign Watcher (auto)

```bash
npm run agents:watch   # Devpost auto-fetch + score + queue
npm run agents:sync    # watch + export status.json + validate
```

- Source: Devpost public API (open/upcoming)
- Output: `contests.yaml`, `campaign-action-queue.json`, then `status.json`
- Join recommendations: score ≥ 45 → queue (human approves registration)
- Prize KYC / submit: **manual** (see `CAMPAIGN_PLAYBOOK.md`)
- CI: daily 06:00 UTC via `.github/workflows/agent-sync.yml`

## Trust rules (mandatory)

1. **Verified sources only** — every contest URL must be official (Devpost project page, Luma, Help Center, X post, etc.).
2. **No fabricated names** — never invent contest titles, deadlines, or metrics.
3. **Show timeliness** — use `period` for human-readable dates; use `deadline` for expiry.
4. **Closed ≠ active** — do not list ended submission windows as open contests.
5. **Auto entries** have `auto: true` / `type: devpost-auto`. Manual verified entries are not overwritten by auto-merge.

## Manual edit → export → push

```bash
npm run agents:add -- contest --title "..." --url "https://..." --ongoing
npm run agents:export
npm run agents:validate
npm run build
```

| File | Agent | When to edit |
|------|-------|--------------|
| `contests.yaml` | campaign-watcher | Auto + manual contests |
| `campaign-action-queue.json` | join list | Generated by `agents:watch` |
| `production-log.yaml` | production-recorder | After a documented production session |
| `publish-manifest.yaml` | work-distributor | When a work is published (with real URL) |
| `ideas.yaml` | idea-miner | When scoring a verified public AI case |
